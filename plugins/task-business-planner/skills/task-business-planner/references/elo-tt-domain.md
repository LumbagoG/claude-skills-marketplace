# elo-tt — Глоссарий проекта в простом русском

Этот файл — словарь для нетехнического партнёра. Все ключевые понятия проекта без жаргона. Используй его, чтобы говорить с партнёром на ЕГО языке, а не общими формулировками.

## Что такое elo-tt одной строкой

**Платформа для настольного тенниса**: помогает игрокам найти где играть, отслеживать рост в рейтинге, участвовать в турнирах; помогает клубам управлять расписанием, столами, турнирами и зарабатывать на бронированиях; даёт сообщество через форум и чат.

## Главные роли (персоны)

### 🎾 Игрок (USER)

Самая массовая роль (~80% пользователей). Что делает:

- Находит клубы рядом со своим городом.
- Бронирует столы для игры.
- Участвует в турнирах клубов.
- Видит свой рейтинг ELO и его рост.
- Получает медали за призовые места в турнирах.
- Общается на форуме, в чате тем, в чате турниров.
- Подписывается на любимые клубы — получает уведомления о новых турнирах.

### 🏛️ Клуб (CLUB)

Владелец / менеджер клуба настольного тенниса. Что делает:

- Управляет своим профилем: описание, фото, контакты, расписание работы.
- Заводит столы (физические столы в зале), устанавливает цены за час.
- Принимает бронирования от игроков, видит расписание загрузки.
- Создаёт и проводит турниры (10 разных систем — single elimination, swiss, round robin и т.п.).
- Управляет персоналом — добавляет сотрудников.
- Отвечает на заявки на работу в клубе (staff applications).
- Принимает рассылки уведомлений (новая бронь и т.п.) на email.

### 👨‍🏫 Тренер (Trainer) — фича в роадмапе

Игрок-плюс, который дополнительно даёт индивидуальные тренировки. Сейчас в плане разработки (см. wiki/tasks/2026-04-29-001-trainers-integration.md). Будет:

- Подавать заявку на тренерство → админ платформы одобряет.
- Привязываться к клубам.
- Выставлять цену за час.
- Принимать заявки на тренировки от игроков.
- Получать отзывы.
- Иметь свою личную страницу.

### 👮 Администратор / Суперадмин (ADMIN / SUPERADMIN)

Сотрудник платформы. Что делает:

- Рассматривает заявки на регистрацию клубов.
- Модерирует форум (репорты, забаненные слова, скрытие постов).
- Управляет ролями пользователей.
- Видит метрики платформы.

### 👤 Гость

Посетитель сайта без регистрации. Может смотреть форум (только чтение), смотреть клубы, видеть профили игроков, видеть результаты турниров. Не может бронировать, писать, участвовать.

## Главные сущности (то, что есть на платформе)

### 🏢 Клубы

Заведения настольного тенниса. У клуба есть: название, город, адрес, телефон, описание, фото, расписание работы (по дням недели), столы, сотрудники, турниры, бронирования.

Каждый клуб — это аккаунт с ролью CLUB. Регистрация клуба идёт через заявку, которую рассматривает администратор платформы.

### 🎾 Игроки

Зарегистрированные пользователи. Профиль: ник, фото, город, рейтинг ELO, статистика (сколько турниров сыграл, сколько побед, peak rating), медали, активная рука (правша/левша), стиль игры.

### 📊 ELO-рейтинг

Числовая оценка силы игрока. Стартует с 1000. После каждого матча в турнире рейтинг победителя растёт, а проигравшего — падает. Чем сильнее соперник, тем больше изменение. Это стандартная шахматная формула, адаптированная под настольный теннис.

Игрок имеет три рейтинга: общий, для правой руки, для левой руки (для амбидекстров).

### 🏓 Столы

Физические столы в клубах. У каждого: название, фото, цена за час (в рублях), активен ли. Игроки бронируют столы на конкретное время.

### 📅 Бронирование

Запись игрока на стол на определённое время. Статусы: ожидает, подтверждено, отменено, завершено.

Каждый игрок имеет «индекс надёжности» (trust index, базово 100). За завершённые брони он растёт, за отмены без предупреждения — падает. Низкий trust index = клуб может отказать в брони.

У клуба настраивается «правило бесплатной отмены» (например, 6 часов до начала). Если отменить позже — индекс падает.

### 🏆 Турниры

Соревнования, организуемые клубами. Атрибуты: название, дата начала, максимум игроков, число столов, минимальный/максимальный рейтинг для участия, взнос за участие, турнирная система.

10 турнирных систем: одиночное выбывание, двойное выбывание, круговая система (round robin), швейцарская система, групповой этап + плей-офф, и т.д.

Жизненный цикл: черновик → регистрация → начался → идут матчи → завершён.

Из турнира выпадает «турнирная сетка» (bracket) — иерархия матчей. Победители получают медали (золото / серебро / бронза).

### 📝 Шаблоны турниров

Клуб может сохранить настройки турнира как шаблон, чтобы быстро создавать похожие турниры повторно.

### 💬 Форум

Сообщество. Категории → темы → посты. Темы могут быть с опросами. Посты можно лайкать, помечать (репорты), реакции. Есть полнотекстовый поиск, теги, упоминания (@nickname), закладки.

Каждая тема имеет свой чат (отдельный от форум-постов) — для быстрых обсуждений в реальном времени.

Каждый пользователь имеет «репутацию форума» — она растёт за полезные посты, падает за нарушения. Низкая репутация = частичный mute (не может писать).

### 💬💬 Чат

Реалтайм-сообщения. Поддерживает: реакции эмодзи, упоминания, файлы вложениями, индикатор «печатает», прочитано / не прочитано, редактирование, удаление, репорты, виртуализированный список (можно листать тысячи сообщений).

Используется в форум-темах. В будущем — для личных сообщений между игроками, между тренером и учеником.

### 📩 Уведомления

Два канала: email + push в приложении (через WebSocket — обновляется в реальном времени без перезагрузки).

События, которые шлют уведомления:
- Новый турнир в подписанном клубе
- Сетка турнира сгенерирована
- Твой матч готов (соперники определены)
- Соперник ввёл результат матча — ты подтверждаешь
- Спор по счёту матча
- Новая бронь твоего стола (для клуба)
- Заявка на работу в клубе одобрена/отклонена

## Что технически нельзя или сложно (high-level — для здравого смысла)

| Хотелка | Реальность |
|---------|------------|
| «Пусть пользователи сами что угодно настраивают» | UI-конструкторы — большой проект. Лучше сначала фиксированные функции, потом customization. |
| «Пусть это работает офлайн» | Веб-приложение требует интернета. PWA даёт частичный офлайн (просмотр, не действия). |
| «Сделаем мобильное приложение» | Отдельный проект на 6+ месяцев. Сейчас сайт хорошо работает на телефоне. |
| «Подключим Apple Pay» | Платежи в России сложны после 2022 — нужно отдельное обсуждение. |
| «Импортируем игроков из другой системы» | Зависит от формата. Excel — да, чужой API — отдельный разговор. |

Это не значит, что эти вещи невозможны — просто они требуют отдельного большого решения. Если партнёр просит такое — фиксируй как business-draft с приоритетом «nice to have», разработчик потом скажет, реалистично ли это в текущем квартале.
